# Intercompany Auto Flow — Odoo Module

## Overview

This module automates intercompany procurement when a company sells a product it does not have in stock. It handles the full document and stock move chain automatically.

---

## The Problem It Solves

**Scenario:**
- Company **A** receives a customer Sales Order for Product X
- Company **A** has 0 units in stock
- Company **B** has 50 units available

**Without this module:** A user must manually create a PO in A, a matching SO in B, and manage the stock transfers.

**With this module:** Confirming the SO in A triggers everything automatically.

---

## Automated Flow

```
Customer → [SO: Company A] → shortage detected
                              ↓
              [Search sibling companies for stock]
                              ↓
              Company B found with sufficient stock
                              ↓
     ┌────────────────────────┴───────────────────────┐
     │ Company A (Buyer)         Company B (Supplier)  │
     │  PO created ←──────────── SO created            │
     │  PO confirmed              SO confirmed          │
     │                                                  │
     │  Stock Picking IN ←──── Stock Picking OUT        │
     │  (B→A transit)           (B internal → transit) │
     └──────────────────────────────────────────────────┘
                              ↓
     Company A stock replenished → fulfills Customer SO
```

## Installation

1. Copy the `intercompany_auto_flow` folder into your Odoo `addons` directory.
2. Restart the Odoo server.
3. Enable **developer mode** in Settings → General Settings.
4. Go to **Apps**, search for *Intercompany Auto Flow*, and install.

**Dependencies:** `stock`, `purchase`, `sale`, `sale_stock`, `purchase_stock`, `account`

---

## Configuration

### Step 1 — Enable on each participating company

Go to **Settings → Companies → [Select Company] → Intercompany Auto Flow tab**

Enable the toggle for **every company** that should participate (both buyers and suppliers).

| Setting | Description |
|---|---|
| Enable Intercompany Auto Flow | Master switch for the company |
| Auto-Confirm Intercompany Sales Orders | Confirm SO in supplying company automatically |
| Auto-Confirm Intercompany Purchase Orders | Confirm PO in buying company automatically |
| Search All Companies for Stock | Search all enabled companies (recommended) |
| Allowed Supplier Companies | Restrict to specific companies (optional) |
| Intercompany Partner | Partner record representing this company (auto-set) |
| Intercompany Route | Optional stock route for PO lines |

### Step 2 — Set up partners

Each company must have a **partner record** in every other company's contact list. Odoo creates these automatically when you create companies — verify they exist under Contacts.

### Step 3 — Set up pricelists (optional but recommended)

For accurate intercompany pricing:
- Add **vendor pricelists** on products: Sales → Product → Purchase tab → add the supplying company's partner with a price.
- The module falls back to `standard_price` if no vendor pricelist entry exists.

### Step 4 — Set up intercompany routes (optional)

If you use multi-step routes, assign your intercompany route in the company settings. This ensures stock moves follow the correct warehousing path.

## How It Works — Code Walkthrough

### Trigger (`sale_order.py → action_confirm`)
When a Sale Order is confirmed, if the selling company has `intercompany_auto_flow = True`, `_trigger_intercompany_flow()` is called.

### Stock Check (`_get_local_available_qty`)
For each SO line with a storable product, the module checks the unreserved on-hand quantity across all **internal locations** of the selling company.

### Supplier Search (`_find_supplier_company`)
- Queries all sibling companies with `intercompany_auto_flow = True`
- Returns the first company that has enough stock to cover the shortage
- Falls back to the company with the most available stock if none can fully cover

### Document Creation (`_create_intercompany_pair`)
1. **Purchase Order** created in buying company (vendor = supplying company's partner)
2. **Sales Order** created in supplying company (customer = buying company's partner)
3. Both are confirmed (if auto-confirm is on)
4. Stock pickings are generated by Odoo's standard flow

### Logging (`intercompany.flow.log`)
Every flow is recorded with:
- Buying/supplying company
- Product and quantity
- Links to the PO and SO
- All stock moves (in and out)
- Status: `draft` → `done` or `error`

## User Interface

### Smart Button on Sale Orders
Confirmed SOs that triggered intercompany flows show an **IC Flows** smart button linking to all related flow logs.

### Flow Logs Menu
**Inventory → Intercompany Flows → Flow Logs** — full list of all flows with filtering by company, product, status, and date.

---

## Edge Cases Handled

| Scenario | Behaviour |
|---|---|
| Local stock fully covers the order | No intercompany flow triggered |
| No sibling company has stock | Warning posted on SO chatter; flow skipped |
| Sibling has partial stock | Flow created for available qty; shortfall logged |
| Exception during PO/SO creation | Error logged to `intercompany.flow.log`; chatter message posted |
| Service/consumable products | Skipped (only `type == 'product'` storable goods trigger the flow) |

## Multi-Company Security Notes

- All intercompany document creation uses `sudo()` scoped with `.with_company(...)` to ensure records are created in the correct company context.
- The `ir.model.access.csv` grants read access to all internal users and full access to stock managers.
- No `ir.rule` bypass is introduced — ensure users have multi-company access configured correctly.
